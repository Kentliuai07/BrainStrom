# ② 开发总蓝图文档

> 把整套技术栈串成「一个会运作的系统」。看完你会懂：使用者点一下，资料和 AI 在前后端怎么跑一圈。
> v1 上架范围 = **市集之前**（Step 0–9，不含 Step 10 社群市集）。

## 一、系统全景图（前 → 后 → AI，跑一圈）

```
┌─────────────── iPhone App（SwiftUI）───────────────┐
│  画面：首页 / 笔记（自由速记↔AI结构化）/ 对话 / 付费   │
│  本地：SwiftData（先存本地，离线也能写）              │
└───────────────┬───────────────────────────────────┘
                │ HTTPS（带登入 token）
                ▼
┌─────────── Supabase（资料层）──────────────┐
│  Auth（Sign in with Apple）                │
│  Postgres：users/systems/blocks/embeddings │
│  pgvector：全局 AI 的「意思相近」搜寻         │
└───────────────┬───────────────────────────┘
                │ App 直接读写资料；AI 代理也连这里做向量搜寻
                ▼
┌──────── Fly.io（AI 算力层 · 常驻不冷启动）────┐
│  AI 代理：所有 Claude 呼叫从这里走（串流）     │
│   └─ 锁着 Claude 金钥、GitHub token          │
└───────┬───────────────┬─────────────────────┘
        ▼               ▼
     Claude          GitHub API
   (结构化/对话/分析)  (读 commit)

付款独立走：iPhone ↔ StoreKit 2 ↔ App Store；后端用 App Store Server API 验证
```

**一句话因果**：App 只画画面、存本地、发请求；**资料真相在 Supabase（Postgres + pgvector）**；**所有秘密和 AI 都在 Fly.io 的 AI 代理里**（为什么独立出来：实测 Supabase 当 LLM 云端会慢，Fly.io 常驻不冷启动、串流更顺）。

## 二、三条主干流程（看懂就懂整个 App）

**A. 写笔记（无 AI）**
点新系统 → SwiftData 先存本地（离线也行）→ 有网时同步到 Postgres（后写覆盖）。
→ 关系：前端负责手感与离线；后端只当「云端备份与真相」。

**B. AI 结构化（核心）**
按「结构化」→ App 把整则文字送 Fly.io AI 代理 → 代理调 Claude（tool_use 强制照 20 卡模板、找不到留空）→ 校验通过就把卡片存进 Supabase 的 `blocks` → 串流回 App 显示。
→ 关系：前端给「乱写的输入 + 一个按钮」；AI 代理保证「输出干净、不乱掰」。

**C. 全局 AI 找回（你最在意的）**
每次结构化后，后端顺手把这则压成「摘要 + 向量」存进 `embeddings`。
你问「那个发票点子在哪」→ 先在 pgvector 找意思相近的几则 → 再交给 Claude 回答 + 指出在哪则。
→ 关系：前端就一个对话框；后端的「记忆」全靠摘要+向量这套。

## 三、v1 要做到哪（范围锁定）

✅ 进 v1：Step 0 验收页、1 地基+登入、2 自由速记、3 结构化、4 模组、5 单笔记AI、6 全局AI、7 Git进度、8 媒体（**只做连结 YouTube/GitHub**）、9 付费+公开/私密上线。
🚫 不进 v1：**语音转写**（延后 v2 之后）、Step 10 社群/市集（避开 UGC 审核风险，上架更稳）。

> 也可以更激进：**最小可上架 = Step 0–6**（笔记+结构化+全局AI 就是完整产品），把 7/8/9 当上架后的更新。这点你定。

## 四、资料怎么流（关键表）

```
users        谁
systems      一个系统/专案（标题、公开私密、模式free/structured、版本）
blocks       系统里的每张卡/每段（type=20种之一、position、payload）
embeddings   每个 system 的摘要向量（全局搜寻用）
memberships  会员状态（免费/PRO、到期日）→ 后端据此开关功能
repo_links   绑定的 GitHub 仓库 + 进度快照（Step 7）
attachments  语音/影片/连结（Step 8）
```

## 五、安全与上架（从第一天就内建，不要事后补）

- **金钥永不进 App**：Claude/GitHub 全锁在 Fly.io 的 AI 代理。
- **权限在后端把关**：私密笔记、进阶模组、进阶模型——前端的锁只是视觉，真正的闸在后端看 `memberships`。
- **上架必备**：App 内「删除帐号」、隐私政策、隐私标签、AI 内容标注、Sign in with Apple、加密声明=YES。
- **市集延后**：v1 没有别人上传的内容 = 没有审核地雷，过审率最高。
