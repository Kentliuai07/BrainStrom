# ③ 开发步骤详解 + Workflow 策略（白话版）

> 给国中生也看得懂。结构：**三大阶段 → 步骤 → 小步骤 → 更细的步骤**。
> v1 上架 = 做到「市集之前」。语音、市集延后到 v2 之后。
> 先给你看，等你指示再开工。

## 怎么读这份
- **阶段** = 一大段时间的目标。
- **步骤** = 一个完整功能。
- **小步骤** = 这个功能要分几块做。
- **更细步骤** = 每块具体动哪些手。

## 三阶段总览

| 阶段 | 目标 | 含步骤 | 做完你会有 |
|---|---|---|---|
| 一 · 看得到 & 能用 | 地基＋能写笔记 | Step 0–2 | 一个能登入、能写、能存的笔记 App |
| 二 · 变聪明 | AI 上身 | Step 3–7 | 会结构化、会聊天、会跨笔记找、会看 Git 进度 |
| 三 · 能上架 | 收费＋送审 | Step 8–9 | 能设公开/私密、能订阅、能上 App Store |
| （v2）社群 | 市集 | Step 10 | 别人能逛、Remix、买卖（之后做） |

> **可以提早上架**：做完阶段二（到 Step 6 或含 Git 7）其实就是一个完整产品，能先上架收反馈；阶段三是把「收费＋私密」补齐。停在哪你定。

# 阶段一 · 看得到 & 能用（Step 0–2）

## Step 0 · 验收页（你的观察窗）
- **小步骤 0.1 后端开一个唯读状态接口**
  - 回传：系统数、最近一笔时间、各功能灯号（DB/结构化/全局AI…）。
  - 不碰隐私内容，只回「数字与状态」。
- **小步骤 0.2 做一页 iOS 样式 HTML 仪表板**
  - 沿用雏形的主题色票（黑曜石/亲和）、卡片、缺角风格。
  - 每个功能一盏灯：做好就亮。
  - 自包含、字体内嵌，你用浏览器打开就看得到。
- **小步骤 0.3 部署成一个网址**
  - 放 GitHub Pages 或 Fly.io，给你一条连结，随时刷新看进度。

## Step 1 · 地基＋登入（无 AI）
- **小步骤 1.1 建资料库**
  - 建表：users、systems、blocks、memberships。
  - 设「每个人只能看自己的资料」的规则（RLS）。
- **小步骤 1.2 Sign in with Apple 登入**
  - App 端：苹果登入按钮 → 拿到身分。
  - 后端：换成我们的登入凭证，存进手机钥匙串。
  - **加一个「删除帐号」按钮**（上架硬性要求，先做）。
- **小步骤 1.3 主题切换**
  - 把雏形两套色票变成 App 的浅/深色，能即时切。

## Step 2 · 自由速记能存能读（无 AI）
- **小步骤 2.1 系统清单（首页）**
  - 读自己的 systems，画成卡片清单。
  - 「＋」新增一个系统。
- **小步骤 2.2 自由速记编辑**
  - 打开一则就能打字；内容存进 blocks（纯文字）。
- **小步骤 2.3 离线也能写 + 自动同步**
  - 先存手机（SwiftData），有网再上传。
  - 同一笔以「最后存的为准」处理冲突。
  - 验收页亮起「DB / 读写」灯。
- **小步骤 2.4 底部工具列 + 加模组旋钮（UI 这阶段就做好）**
  - 用 SwiftUI 手势把旋钮做出来：能转、能吸附、点中心选取。
  - 阶段一就能真的插入「基础块」：文字段落、待办、标题。
  - 需要 AI 的模组（心智图/分析…）先显示占位，阶段二接上 AI。
- **小步骤 2.5 顶部「自由速记 / AI 结构化」切换（这阶段就做好）**
  - 两个分页都做出来、能切换。
  - 「自由速记」完整可用；「AI 结构化」视图先做成空状态（「还没整理，去阶段二接 AI」），Step 3 再灌真内容。

> 范围调整（你拍板）：**阶段一一次盖好完整前端骨架**——含底部工具列、加模组旋钮、双模式切换。
> 但「旋钮插入 AI 模组」「结构化的 AI」属于阶段二才接；阶段一它们是**能动的空壳**。

# 阶段二 · 变聪明（Step 3–7）

## Step 3 · AI 结构化（产品灵魂）
- **小步骤 3.1 在 Fly.io 架 AI 代理**
  - 一台常驻小机器，锁住 Claude 金钥。
  - 提供一个 `/structure` 接口。
- **小步骤 3.2 设计「20 卡模板」契约**
  - 用 tool_use + JSON schema 强制 Claude 照固定格式回。
  - 规则：只能根据笔记内容、找不到留空、不准编。
- **小步骤 3.3 校验 + 重试**
  - 回来先检查格式对不对；不对就重试或升级到更强模型。
  - 通过才把卡片存进 blocks。
- **小步骤 3.4 前端显示 + 串流**
  - 按「结构化」→ 一张一张卡浮现（串流）。
  - 顶部切换：自由速记 ↔ AI 结构化。

## Step 4 · 模组化（旋钮真的能加）
- **小步骤 4.1 左下角旋钮 UI**
  - 用 SwiftUI 手势做旋转、吸附、点中心选取（取代网页的 GSAP）。
- **小步骤 4.2 真插入**
  - 选一个模组 → 后端在 blocks 新增一笔 → 列表刷新。
- **小步骤 4.3 补齐缺的模组卡**
  - 例如「表格化、AI 搜寻」等卡的样子与资料。

## Step 5 · 单笔记 AI 对话（第 1 层记忆）
- **小步骤 5.1 对话接口**
  - `/chat/note`：把这一则的全文当背景丢给 Claude。
- **小步骤 5.2 对话 UI**
  - 笔记底部聊天浮层；显示历史、建议追问。
- **小步骤 5.3 串流回覆**
  - 一个字一个字跳出来，体验顺。

## Step 6 · 全局 AI 跨笔记找回（第 2 层记忆）
- **小步骤 6.1 做「摘要 + 向量」**
  - 每次结构化后，顺手把这则压成摘要，转成向量存进 embeddings。
- **小步骤 6.2 向量搜寻**
  - 你问一句 → 先在 pgvector 找意思相近的几则。
- **小步骤 6.3 回答 + 指出处**
  - 把找到的几则交给 Claude → 回答并告诉你「在哪一则」。
  - 免费上限 50 则，避免脑容量爆。
  - 验收页：输入模糊描述 → 显示命中的系统（核心测试）。

## Step 7 · 连 Git ＋ AI 判断进度 ＋ 进度图
- **小步骤 7.1 GitHub 授权**
  - 用 GitHub 登入授权，token 锁在 Fly.io。
- **小步骤 7.2 读仓库**
  - 抓 commit、档案树、最近活跃度。
- **小步骤 7.3 AI 比对算进度**
  - 拿 Step 3 的「建置步骤」当清单，逐条判断做了没。
  - 算粗略完成度（先求有、不求精）。
- **小步骤 7.4 进度图**
  - 环形百分比 ＋ 步骤打勾 ＋ 活跃度；新增 devProgress 卡。

# 阶段三 · 能上架（Step 8–9）

## Step 8 · 媒体（v1 只做连结）
- **小步骤 8.1 连结自动成卡**
  - 丢 YouTube/GitHub 连结 → 抓标题、星数、缩图，变一张卡。
- **小步骤 8.2 自动摘要**
  - AI 把连结内容整理成几句重点。
  - （语音转写：延后 v2 之后，不在这里。）

## Step 9 · 付费 ＋ 公开/私密上线
- **小步骤 9.1 苹果内购（StoreKit 2）**
  - App 端：订阅画面、购买流程。
  - 后端：用 App Store Server API 验证收据，写进 memberships。
- **小步骤 9.2 权限闸门（后端把关）**
  - 私密笔记、进阶模组、进阶模型：后端看会员状态才放行。
  - 前端的锁只是视觉，真闸在后端。
- **小步骤 9.3 上架合规清单**
  - 删帐号、隐私政策、隐私标签、AI 内容标注、加密声明=YES。
  - 没有市集 = 没有别人内容要审 = 过审最稳。
- **小步骤 9.4 送 TestFlight → App Store**
  - GitHub Actions + Fastlane 自动打包上传。

# 策略附录 · 本 Workflow 怎么用「代理大军」开发

> 核心打法：**每一个小步骤，都先派侦察兵摸清楚，再派工兵动手，最后派验收兵点灯。**

## (a) 探勘代理大军（侦察兵 · 只看不动手）
- 任务：动工前先把这一步的「正确做法、官方文件、坑」摸清楚。
- 例子：
  - 「Sign in with Apple 接 Supabase 的正确步骤与坑」
  - 「StoreKit 2 收据验证怎么做」
  - 「tool_use 强制 JSON 的稳定写法」
- 产出：一页「事实 + 最佳做法 + 雷区」给工兵照着做。

## (b) 工作代理（工兵 · 真的写）
每一步派对应的工兵，分工：
- **前端代理**：写 SwiftUI 画面与互动。
- **后端代理**：建表、写接口、Fly.io 部署。
- **AI 代理**：写提示词、20 卡模板、RAG 检索。
- **整合代理**：把前端↔后端↔AI 接起来。
- **验收代理**：写测试、更新验收页那盏灯。

## (c) 设计指令 与 执行指令（给每个工兵的两张纸）
- **设计指令（要做什么）**：从这三份文档来 —— 要盖的功能、资料格式、契约、限制。
  - 例：「结构化要回这 20 个栏位的 JSON；找不到留空；金钥不进 App。」
- **执行指令（怎么做＋怎么验）**：写码规范＋自检。
  - 必须能编译、必须写一个小测试、必须更新验收页、不准把金钥放 App、做完 commit。

## 每个小步骤的固定循环
```
侦察兵摸清楚 → 设计指令 → 工兵动手 → 验收兵自检 → 验收页点亮一盏灯 → 你过目
```
- 一次只点亮一盏灯，进度看得见、好回头。
- 你随时能在验收页看到「现在做到哪」。
