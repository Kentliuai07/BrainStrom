// 配方 D：TERMINAL

  • 底色：GitHub Dark 系 #0D1117，面板 #161B22——開發者每天住的顏色，零學習成本的「專業感」。
  • 系統感：頂欄是檔案路徑麵包屑＋同步綠點；中繼資料用 mtime / chars / rev；底部 status bar（同步狀態/系統名/字數/AI 待命）= IDE 簽名。
  • 手感（手繪的替代）：mono 字元就是這個風格的「手寫」——[x] 待辦、// PINNED 註解語法、❯ 提示符、▰▱ 進度。
  • 科技模組：模組卡=終端輸出面板（紅黃綠窗點＋指令行＋游標閃爍）；直條按鈕=機械鍵帽（1px 立體＋按下位移 2px，遊戲趣味藏在按鍵手感）。
  • 缺角簽名保留：模組卡右上切角、直條左上切角。

    • // Light（按右上切換）

  • GitHub Light：白底 #FFF＋灰面板，綠降為 #1A7F37 保對比——像日間的編輯器主題。

    • // 評分時看什麼

  • mono 元素的密度——夠味還是太「工程文件」？鍵帽按下去的感覺？status bar 是加分還是占空間？
