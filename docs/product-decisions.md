## 2026-06-11 · 内容模型 v3「活文件模型」（取代「双模式 free/structured」与「三层模型」）

- **一份笔记 = 一份会不断进化的活文件** = 一串有顺序的块（段落块＋模组块）。「**文章**」与「**卡片**」是同一串块的两个视图，不存两份、天生同步。**没有永久并存的「自由速记原始层」**——最早的原稿就是版本历史第 1 版。
- **两个 AI 按钮**：「优化文字」（整理排版文笔，仍是文章；每次都问「要不要分主题加小标」）／「卡片结构化」（归组成卡，共用优化管线）。
- **AI 可改可删**未钉选的块（含手动加的）；**钉选（pinned）的 AI 永不碰**；模组卡内容 AI 永不动。
- **安全网**：每步操作（AI 操作、改卡、删卡、合并、拆分）自动存档；**上一步/下一步不限次数**、版本快照无限保留；回旧版只做整篇还原（不做捡回片段）；打字不算步。
- **省钱**：内容指纹没变不叫 AI；没变的段落一个字不重送（兼防 AI 反复重写造成内容走样）。
- 本节**取代**下方旧决策中「双模式(free/structured)」的语意（mode 改为 `docState` 生命周期）；详细规格见 `docs/阶段二开发文档-变聪明-AI功能.md` §0.2、§1.2。

## 2026-06-09 · 新功能：连接 Git 仓库 + AI 判断进度 + 进度可视化

- 使用者可以**连到自己正在开发的 Git 仓库**（GitHub/GitLab）。
- AI 读取仓库（commit 历史、档案结构、分支、PR），**判断目前开发进度**。
- 把进度对照这则笔记「AI 结构化」里的**建置步骤**，算出完成度，并显示一张**可视化进度图**（环形/时间轴/步骤勾选）。
- 这是一个新的**模组块**：`devProgress`（开发进度），也放进旋钮模组清单。
- 需要：GitHub OAuth 授权、读取仓库 API、AI 比对「计划 vs 实际」、进度快照储存。
- 排进开发进程的 **Phase 7**（核心 AI 跑通后再做，因为要先有「建置步骤」当比对基准）。
- 详见 `roadmap.md` 第 7 步、`backend-design.md`。

## 2026-06-09 · 图标、模组入口、双模式

### 1. 图标：不用 emoji，用高质感图标库
- **所有符号一律不用字体 emoji**（💡🔄… 这种），改用**高质感图标库**里的矢量符号，风格统一。
- HTML 稿用 **Lucide**（开源、线条统一、质感高）；将来原生 iOS 用 **SF Symbols**。
- 所有图标都**为氛围开发定制**（分类、模组都用一套统一的符号语言）。

### 2. 模组入口：左下角工具按钮
- 笔记**左下角**放一个**工具按钮**，按下去弹出**模组**列表。
- 这些「符号 = 模组」，就是之前讨论过的、可以加进笔记里的东西（心智图 / 影片 / GitHub / 代码 / AI 分析 …）。
- 全部针对**氛围开发**场景。

### 3. 同一个系统/专案内的两种模式（核心）
一条笔记（一个系统主题）里同时存在两种视图，用户可随时切换：

**① 自由记录模式（Free）**
- 想到什么写什么，像 Apple Notes 一样，零负担。
- 这是「输入面」，可手动用左下角工具插模组。

**② AI 整理模式（Organized）**
- 用户**按下按钮**后，AI 把**新写的内容整理进去**。
- 需要为这种模式设计一套**格式化的呈现方式**（结构化版面：按分类/章节归位、抽出待办、生成心智图、识别工具…）。
- 这是「整理/回看面」，干净、可导航。

**交互**：每次进入这个系统，用户可以选择
- 直接记一句话（进自由模式），或
- 查看之前 AI 整理过的格式化版本（进整理模式）。

**与记忆架构的关系**：负责「整理」的就是这本笔记的**第 1 层主题专属 AI**（见 `memory-architecture.md`）。整理 = 把自由文本结构化为分类 + 模组。

**待验证**：自由模式与整理模式之间，是「同一份内容的两个视图」，还是「整理会生成一份新的结构化副本」？建议 MVP 里先做「同一份内容、两个视图」，整理结果可被用户再编辑。
