## 1. 设计授权（2026-06-11 使用者拍板，取代旧视觉规范）

- **不再沿用** mvp HTML 的样式与旧规格书的视觉识别（缺角卡片、黑曜石/亲和色票不再是硬性要求）。
- 由你**全新设计**一套视觉语言：**开发者会喜欢的工业风，带优雅与艺术感**——参考方向（非硬性）：精密仪器感、等宽字体点缀、细线网格、克制的高对比强调色、深色优先、微妙的材质与发光、像高级 IDE 与精品硬件的混血。
- **设计自由的边界（不可越）**：
  - 样式自由，**交互行为不准自由**——每颗按钮长怎样随你，按下去发生什么必须照《功能对等清单》逐条一致。
  - 必须保留的体验骨架：文章/卡片两视图同源切换、点哪段改哪段、命名态 gate、⚡ 助攻胶囊、聊天面板＋提议按钮、↶↷ 无限撤销、钉选标记、四态（载入/空/错误/正常）。
  - 模组选择器维持「右缘竖直条＋全屏模糊唤出」的交互模式（视觉重绘随你）。
  - 无障碍与可用性底线：44pt 命中区、Dynamic Type、对比 ≥4.5:1、Reduce Motion 降级。
- 工作法：先用官方前端设计插件产出整套高保真设计稿（工业风），使用者确认后才写画面代码。

## 2. 功能范围（= 网页版现状，一模一样）

照《功能对等清单》全量实作，速览：
- 登入（dev 流程，Apple 登入阶段三才接真）
- 首页：笔记卡清单、＋ 新建、设定入口
- 笔记页：命名态 gate（先取名才能写＋先随便取＋空壳离开自动丢弃）、文章视图段落编辑器（点哪段改哪段/续写区切块/📌 钉选/▲▼/删除/2000 字提示）、卡片视图（就地编辑/钉选/删除/空状态）、文章⇄卡片切换（未卡片化禁用）
- 四条 AI 流（**全部接真后端**）：💬 聊天（流式/token/引用徽章/停止）、⚡ 点子助攻（kickoff 教练开场/重播/✕/总开关）、✦ 优化（分主题确认框/hash gate 省钱/安全阀）、▦ 结构化（逐卡浮现）＋聊天提议按钮（edit_text→applyEdit、structure、find_* 🔒即将推出）
- ↶↷ 无限 Undo/Redo（指针法快照）、版本语意、软删
- 设定页：主题（你的新双主题或明暗模式）、登出、删帐号、点子助攻开关
- **不做**：付费墙、增量结构化/全局找回/GitHub 进度（后端未上线，UI 可留位）

## 3. 后端（已上线，直接接真的）

- Base URL：`https://brainstrom-ai.fly.dev`（健康检查 `GET /ai/health`）
- 端点：`/ai/chat/note`（含 kickoff＋proposal）、`/ai/optimize`（含 instruction 模式）、`/ai/structure`（mode=full）——请求/SSE 形状照《整合契约与算法规格》第 1、2 节
- 鉴权：`Authorization: Bearer <AUTH_TOKEN>`——token 由使用者贴给你，放 `Config.xcconfig`（**不进 git**，附 `Config.example.xcconfig`）；Anthropic 金钥永远只在 Fly，App 内零金钥
- **关键架构裁定**：资料真相在本机（SwiftData），server 无状态——hash gate、diff、安全阀、合并套用、快照/Undo **全部在客户端实作**，算法照《整合契约与算法规格》第 3 节逐函式移植（行为必须与网页版一致，差一点就会出现「iOS 扣钱网页不扣」级别的分歧）

## 4. 工程规范（沿用旧规格书 §7/§8，不变）

- 资料夹（Feature-first）：`App/ Features/{Auth,Home,Note,Settings}/ Core/{Models,Services,Networking,Persistence,DesignSystem}/ Shared/ Resources/ Tests/`
- 抽象层：View → @Observable ViewModel(@MainActor) → Service 协议 → Live/Stub 实作；DI 走组合根；View 禁止直连网络；AI 串流以 `AsyncThrowingStream<AIEvent,Error>` 暴露（AIEvent 镜像 SSE 事件枚举）
- 编译规范：Swift 6 严格并发、warnings as errors、SwiftLint（force_unwrap 禁/档 ≤400 行/函式 ≤60 行）、SwiftFormat、iOS 17+、conventional commits、每画面 #Preview（双主题）
- 性能预算：包体 ≤15MB、零第三方依赖原则、冷启 ≤400ms、60/120fps、内存 ≤150MB、主线程零磁盘 IO
- 测试：算法纯函式必须有单元测试（用《整合契约与算法规格》里的行为描述当用例；hash/diff/安全阀至少各 3 例）、SSE 解析 fixture 测试

## 5. 工作循环（必守）

1. **动工前盘点**：读完四份文档后，先输出一页「理解摘要」（画面清单/服务方法/接后端方式/算法移植清单/设计方向 moodboard 文字版）给使用者确认，OK 才动工。
2. 设计稿包（工业风全套）→ 使用者确认 → 工程骨架（资料夹/DI/Lint/xcconfig/tokens/AIEvent）→ 使用者过目 → 逐画面实作（每画面：四态＋双主题 Preview＋自检 §10 清单 → 使用者验收 → 下一画面）。能接真后端的画面当场接真的。
3. 分支纪律：**全部工作在 `ios-native` 分支**，只动 `ios-app/` 与本节允许的 xcconfig 范本；`web/`、`docs/`、`server/` 唯读。`git push -u origin ios-native`。不开 PR 除非使用者明讲。
4. 规格疑义 → 先查《功能对等清单》对应条目 → 再看 web/src 源码行号佐证 → 仍无解就输出「契约变更申请」等使用者裁决，不准自行发明。
5. 全程国中生白话文沟通，每则回覆结尾附白话总结。

## 6. 验收（最终标准）

逐条勾《功能对等清单》＋旧规格书 §10 工程清单；AI 四条流用真后端实测（聊天引用内容、优化省钱提示、结构化逐卡浮现、提议落地可 Undo）。使用者会拿网页版对照行为，**两边按同一个按钮必须发生同一件事**。
