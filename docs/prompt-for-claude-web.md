# 给 Claude 网页版的制作提示词 · 氛围开发笔记（BrainStrom）

> 直接把下面「===」之间的内容整段复制，粘贴到 Claude 网页版即可。

===

帮我做一个 **iOS 手机 App 的可交互前端原型**，产品叫 **BrainStrom（氛围开发笔记）**。

## 交付物
- 做成**一个单文件、可直接在浏览器打开运行的原型**（HTML + Tailwind CDN + 一点 JS；或单文件 React 也行）。
- 手机尺寸，带 **iPhone 外框**（状态栏、灵动岛、底部 Home 条）。
- 用**假数据**，示例项目用「**PDF 问答助手**」。

## 这个产品是什么
给「氛围开发者」（vibe coder，很多不是程序员、不懂术语的创意人）记录系统灵感的笔记工具。
**每一条笔记 = 一个待开发的系统主题**。用户可以不停往里随手写，AI 负责把它整理成统一结构。

## 视觉风格
- **参考 Apple Notes**：暖白纸面、干净排版、居中日期、大标题、舒适行距、毛玻璃工具栏。
- 中文要用接近**苹方**的字体观感；英文/数字用 Inter 一类。
- **所有图标都用线性图标库（如 Lucide），不要用 emoji。** 风格统一。

## 核心：同一条笔记，两种模式（顶部用 iOS 分段控件切换，要能真的点击切换）

### 模式一：自由记录
- 像 Apple Notes 一样，**想到什么写什么，不用排版、持续输入**。
- 这是「输入面」。

### 模式二：AI 结构化（重点）
- 笔记右侧有一个「**AI 结构化**」按钮，点一下，**整篇笔记被 AI 整理成统一格式**。
- 因为是给氛围笔记用的，所以**每一篇结构化后的格式都一模一样**，固定包含这 7 块：
  - **(a) 系统名称** —— 这个系统/专案叫什么
  - **(b) 技术栈** —— 用到的语言 / 框架 / 平台工具
  - **(c) GitHub 参考** —— 可直接拿来用的开源仓库（让不会写代码的人也能找到现成的）
  - **(d) 开发步骤** —— 从 0 到 1 的步骤清单
  - **(e) 使用者操作说明** —— 终端用户怎么使用这个产品
  - **(f) HTML 样式区** —— 一块可以放 / 预览 HTML 样式片段的空间
  - **(g) 开发重点** —— 可细分为 **UI/UX、前端、后端** 三类的重点提示
- 每一块都做成一张清爽的卡片，带标题和对应图标。

产品逻辑一句话：**用户有一个不用排版、可持续输入的版本；AI 把它转化成上面这个格式化版本。两个版本共存，随时切换。**

## 左下角工具栏（加模组）
- 笔记**左下角**有一个**模组工具按钮**。
- 点击后弹出**模组列表**，用户可以挑自己喜欢的模组插进笔记。
- **MVP 版本里，每个模组都要有名称**（之前只有图标没有名字，现在要把名字放上）。
- 模组列表（都为氛围开发定制）：
  1. 开发逻辑（可生成流程/心智图）
  2. 平台工具
  3. 参考影片
  4. GitHub 开源
  5. 提示词库
  6. AI 分析

## 示例内容（「PDF 问答助手」，给你填充用）
- 自由记录原文（大意）：想做一个上传 PDF 就能直接提问的工具，不用读整份。流程：上传→切块→转向量存库→提问时检索相关段→连同问题丢给 Claude 回答。后端用 Supabase（pgvector），前端对话用 Vercel AI SDK。提示词要强调「只根据检索到的内容回答，找不到就说不知道，绝不编造」。参考一个讲 RAG 的 YouTube 教程和 GitHub 上 langchain 的例子。
- AI 结构化后填进 (a)–(g)：
  - (a) 系统名称：PDF 问答助手
  - (b) 技术栈：Supabase + pgvector、Vercel AI SDK、Claude API
  - (c) GitHub 参考：langchain-ai/langchain（RAG 范例）、supabase/supabase
  - (d) 开发步骤：① 上传切块 ② 转向量存库 ③ 提问→检索→Claude 生成答案
  - (e) 使用者操作说明：上传 PDF → 在对话框直接提问 → 得到带出处的答案
  - (f) HTML 样式区：放一个聊天气泡 + 答案卡片的样式片段占位
  - (g) 开发重点：UI/UX（对话式、答案要标出处）；前端（流式输出）；后端（向量检索准确率、找不到时的兜底）

## 交互要求
- 顶部分段控件「自由记录 / AI 结构化」**可点击切换两个视图**。
- 左下角模组按钮**可点击弹出模组列表**。
- 右侧「AI 结构化」按钮**可点击**，点了就从自由记录视图切到结构化视图（可以用预置好的结构化结果模拟）。
- 注重质感：圆角、留白、柔和阴影、线性图标，整体像一个精致的 iOS App。
