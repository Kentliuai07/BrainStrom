# BrainStrom · 開發路線圖（白話版）

> 用最白話的方式，把整個 App 該怎麼一步步做出來，從頭排到尾。
> 配合 `ios-structure.md`（前端）、`backend-design.md`（後端）、`memory-architecture.md`（AI 記憶）看。
> 原則：**先讓畫面動起來（吃假資料）→ 再一塊一塊把真功能接上去。**

## 十個步驟（照順序做）

**1. 打地基**
建資料庫、登入、後端骨架。就像蓋房子先挖地基、接水電。這步沒畫面，但少不了。

**2. 筆記能存能讀**
做「我的系統」清單 + 打開一則筆記用「自由速記」寫字。重開 App 內容還在。
→ 這時候 App 第一次「真的有用」了。

**3. AI 結構化（產品的靈魂）**
按一下「AI 結構化」，把你亂寫的速記，變成乾淨的 ~20 張卡（系統名稱／技術棧／開發邏輯／GitHub／提示詞…）。
→ 最該先驗證：亂七八糟的輸入，AI 能不能穩定變成漂亮輸出。

**4. 左下角旋鈕真的能加模組**
撥旋鈕選一個模組（例如「提示詞」），筆記裡**真的多一張卡**（現在的雛形只會跳提示，是假的）。

**5. 跟「這則筆記」的 AI 聊天**
針對單一系統提問，AI 只看這一則的內容回答。這是**第 1 層：主題專屬 AI**。

**6. 全局 AI：跨所有筆記找東西**
問「我那個跟發票有關的點子寫在哪？」AI 能跨全部筆記找回來。這是**第 2 層：全局 AI**，也是你最早最在意的功能。
→ 技術上要先把每則筆記做成「摘要 + 向量」才能搜尋。

**7. 連接 Git 仓库 + AI 判斷進度 + 進度圖（你新加的功能）**
把你正在開發的 GitHub 仓库連進來，AI 讀 commit／檔案／分支，**對照第 3 步算出來的「建置步驟」**，判斷你做到哪了，畫一張**可視化進度圖**。
→ 為什麼排在這：要先有「建置步驟」當尺，才量得出「實際做了多少」。

**8. 媒體變真的**
語音真的能錄音轉文字；丟 YouTube／IG／GitHub 連結，自動抓真資料＋摘要。

**9. 付費會員**
接 Apple 訂閱，鎖住「私密筆記」和「進階模組」，PRO 才解鎖。錢的閘門一定在後端把關。

**10. 探索／社群市集**
別人能看你公開的系統，一鍵 Remix、買賣、聯絡作者。這是長線的社群與營收。

## 為什麼是這個順序（相依關係）

- **3 要先於 4、7**：旋鈕要插的卡、Git 進度要比對的步驟，都來自「結構化」。
- **5 要先於 6**：先做單一筆記的 AI，再擴成跨筆記的全局 AI。
- **6、7 是兩大亮點，但都靠前面的地基**：沒有 3 的「步驟」、沒有資料庫，這兩個都做不出來。
- **9、10 最後做**：先把產品做到「好用」，再談「收錢」和「社群」。

## 階段切分（給節奏感）

- **第一階段 · 能用**：步驟 1–4（能寫、能結構化、能加模組）
- **第二階段 · 變聰明**：步驟 5–7（單筆記 AI → 全局 AI → Git 進度）
- **第三階段 · 能賺錢 & 長大**：步驟 8–10（媒體、付費、社群）

## Git 進度功能怎麼運作（白話拆解）

1. 你按「連接 Git」→ 跳出 GitHub 授權（像很多 App 那種「用 GitHub 登入」）。
2. 後端去讀你的仓库：有幾個 commit、改了哪些檔、有哪些資料夾、最近多久動一次。
3. AI 拿「這則筆記的建置步驟（第 3 步產生的）」當清單，一條一條對：這步看起來做了沒？
4. 算出完成度（例如 4 步做完 2 步 = 50%），畫成一張圖：環形百分比 ＋ 每個步驟打勾／未打勾 ＋ 最近活躍度。
5. 之後每次打開可重新整理，進度會跟著你的真實 commit 一起動。

> 風險：每個人專案結構差很多，AI「看 commit 猜進度」會有誤差。MVP 先做「步驟勾選 + 粗略百分比」，別追求太精準。
